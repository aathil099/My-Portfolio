Energy Monitoring System

Designed and implemented a customized IoT-based Energy Monitoring System tailored for the garment industry, utilizing the ESP32-WROOM-32UE microcontroller and CT sensors to ensure accurate, real-time measurement of voltage, current, power factor, and energy consumption.

The system features a robust circuit designed in Altium Designer and integrates seamlessly with ThingSpeak for secure cloud data storage and a SD module is also integrated to prevent data loss. A dedicated web interface displays real-time analytics and visualizations, enabling users to monitor energy usage remotely. Additionally, machine learning algorithms were employed to predict consumption patterns and provide actionable insights, supporting energy optimization and cost reduction efforts.

System Overview

The system is composed of energy monitoring nodes that measure voltage, current, active power, and cumulative energy consumption using the V9881D energy measurement chip. A low-cost energy meter was designed using a non-intrusive current transformer (CT) and an ESP32-WROOM microcontroller, which includes a built-in wireless communication module. An LCD is integrated to display real-time measurements, and an SD card module is implemented to store data locally during Wi-Fi outages, ensuring no data is lost. The microcontroller is powered by a DC power supply equipped with a voltage regulator circuit.

The system is designed to be user-friendly, with a web interface that allows users to monitor energy consumption in real-time. The data is stored in the cloud using ThingSpeak, enabling remote access and analysis. The system also includes machine learning algorithms that predict energy consumption patterns, providing users with insights into their energy usage and helping them optimize their consumption.

In my project, I developed a program using the ESP-IDF development framework, structured in the following manner.

  • Start: Marks the beginning of the program, where necessary header files and dependencies are included.
  • Initialization: This stage sets up all essential hardware components, including the Wi-Fi module, SD card module, and sensors required for data acquisition. The LCD displays a welcome message, indicating that the system has started successfully.
  • Main Loop Start: The core of the program begins here. As is standard in embedded systems, the program enters an infinite loop that continuously monitors and processes data. In this phase, UART transmission and LCD display functionalities are activated, and sensor readings are initiated.
  • Data Collection: The system actively gathers data from the configured sensors and input devices during each loop iteration.
  • Wi-Fi Connectivity Check: The program checks whether the device is currently connected to a Wi-Fi network.
  • Data Handling: If a Wi-Fi connection is available, the system uploads the collected data to the cloud in real time. If Wi-Fi is not available, the data is stored locally on the SD card to ensure no data loss.
  • End of Main Loop: This indicates the end of the current iteration. The program then loops back to the Main Loop Start, maintaining continuous operation.
  • Loop Continuation: This step is inherently managed by the infinite loop structure, allowing the system to function without interruption.
  • Stop: Represents the theoretical endpoint of the program. However, in practical embedded system deployments, the program continues to run indefinitely until the device is manually powered down.

ThingSpeak was integrated into the energy monitoring system due to its user-friendly interface and flexibility. A dedicated channel was created with fields for current, voltage, power, energy, power factor, and frequency. The ESP32-WROOM microcontroller formatted and transmitted data using the ThingSpeak API, ensuring accurate and synchronized updates every 5 seconds. The platform enabled secure data transmission, real-time monitoring, and remote access. Its graphical visualization tools made the measured parameters easy to interpret. Team members accessed raw data via the read API key for analysis and decision-making. Below are screenshots of the ThingSpeak dashboard displaying real-time energy data.

This project successfully developed an IoT-based energy monitoring system specifically designed for the garment industry. By combining reliable hardware and software components, the system provides real-time tracking of essential energy parameters such as voltage, current, power factor, and overall energy consumption. The ESP32-WROOM-32UE microcontroller and CT sensors were selected for their high compatibility and accuracy, ensuring dependable performance in industrial settings. Circuit design was carried out using EasyEDA, followed by hardware assembly on a perf copper board. A draft PCB layout was also created to support future production needs.

Integration with the ThingSpeak cloud platform enabled secure data transmission, remote access, and real-time visualization, adding scalability to the solution. Through regular data updates and a user-friendly interface, manufacturers can monitor energy usage trends, identify inefficiencies, and make informed decisions to optimize consumption and reduce costs. The system underwent extensive testing, confirming its ability to accurately measure and display energy metrics, thus demonstrating its potential to significantly improve energy management in the garment sector.

EMS Breakout Board

Designed and implemented a customized IoT-based Energy Monitoring System tailored for the garment industry, utilizing the ESP32-WROOM-32UE microcontroller and CT sensors to ensure accurate, real-time measurement of voltage, current, power factor, and energy consumption.

The system features a robust circuit designed in Altium Designer and integrates seamlessly with ThingSpeak for secure cloud data storage and a SD module is also integrated to prevent data loss. A dedicated web interface displays real-time analytics and visualizations, enabling users to monitor energy usage remotely. Additionally, machine learning algorithms were employed to predict consumption patterns and provide actionable insights, supporting energy optimization and cost reduction efforts.

DC Power Supply (18650 → Regulated 5 V)

Battery input • Protection • 5 V regulation

The node is powered by 18650 Li-ion cells regulated to 5 V. The design includes input protection, regulation, and decoupling to ensure stable supply for the ESP32 and peripherals.

ESP32-WROOM-32E handles acquisition and connectivity. V9881D + CT measure voltage/current and compute active power/energy. A 16×2 LCD (I²C) shows live metrics; the SD module buffers data when Wi-Fi is down.

The board omits an onboard JTAG to keep the design compact. External wire connectors provide flexible I/O, while a dedicated TTL header enables programming and firmware flashing via an external TTL programmer.

The circuit was designed in Altium Designer and integrates with ThingSpeak for secure cloud logging; an SD module prevents data loss during connectivity gaps. A web UI displays live analytics for monitoring.

Final PCB Assembly

SMD layout • Compact routing • Verified in test

The PCB uses SMD footprints for compactness and reliability. After assembly and verification, the node accurately measured and displayed energy parameters in real time, validating the layout and routing choices.

Matrix Vector Multiplier UART System

This project demonstrates the complete ASIC design flow for a Matrix Vector Multiplier UART system using Synopsys EDA tools. The RTL code was synthesized with the SAED 32nm library in Design Compiler, and the final Place & Route (PnR) was carried out in ICC2. The work emphasizes key steps such as synthesis, timing analysis, and physical layout generation.

RTL Synthesis and Visualization

Design Compiler • RTL Schematic • Synthesis Reports

The RTL was successfully synthesized and visualized in Design Vision. Reports generated include ports, area, power, and timing. For instance, the total design area was approximately 102,264 µm² with a dynamic power of 1.56 mW. The clock period was tuned to 10.5 ns to ensure timing closure.

Synthesis Metrics

  • Ports: 4 I/O ports developed and verified
  • Area: 102,263.87 µm²
  • Power: 1.5619 mW (dynamic)
  • Cells: ~73,806 cells used
  • Timing: All wire delays met required constraints

Place and Route (PnR)

Node placement • Clock routing • Trace inspection

The physical design was completed in ICC2. Node placement and routing were optimized for both {2,2,2,2} and {8,8,8,8} configurations. The clock network was created using the command create_clock -period 10.5 clk, ensuring synchronization across the layout. Final routing inspections confirmed no violations or critical errors.

Final Layout & Validation

Compact floorplan • Verified routing • Error-free design

The final chip layout was successfully generated with all error checks passed. This validates the ASIC flow execution from RTL synthesis to PnR, showcasing practical knowledge of Synopsys toolchains for digital VLSI design.

Digital Stop Watch

Designed and implemented a digital stopwatch system using open-source ASIC tools and the IHP SG13G2 PDK. The project demonstrates the complete digital design flow – from RTL coding and verification to synthesis, place & route, and final GDSII export. The design also explores practical state machine modeling for sequential logic.

System Architecture

Finite State Machine • Sequential Logic • Behavioural Verilog

The stopwatch was modeled as a finite state machine (FSM). Two combinational blocks drive the design: next-state logic and an output decoder. This structure ensures predictable transitions between start, stop, increment, and reset states.

Functional Blocks

Decimal Counter • Seven Segment Controller

A four-digit decimal counter was designed using cascaded 4-bit binary counters. Terminal count signals (TC) reset each counter and drive carry logic for higher digits, enabling precise millisecond counting.

A seven-segment display controller decodes counter outputs and drives the display. The controller was built with combinational logic and tested for readability under multiple states.

Design Requirements

  • Time Range: 0.000 – 9.999 seconds (1 ms resolution)
  • Controls: Start • Stop • Increment • Reset
  • Output: Four-digit 7-segment display
  • Verification: Waveform validation via EDA Playground & Makerchip

ASIC Flow (OSIC + OpenROAD)

RTL → Gate Level → Floorplan → PnR → GDSII

The Verilog design was synthesized using Yosys + ABC, floorplanned and placed with OpenROAD, and routed with FastRoute. Physical verification included DRC/LVS checks via Magic and timing analysis using OpenSTA. The final GDSII was visualized in KLayout, confirming a valid chip layout ready for tape-out.

Outcome

This project highlights digital ASIC design using entirely open-source EDA tools. It successfully demonstrated the full flow of designing, simulating, synthesizing, and implementing a stopwatch system, providing both functional verification and physical layout readiness.

Plant Monitoring System

Developed a low-cost IoT-based smart irrigation system using the ESP32 microcontroller, soil moisture and DHT11 sensors, and a 12V water pump. The system leverages FreeRTOS multitasking and integrates with ThingSpeak cloud to monitor and automate water supply based on real-time soil conditions.

System Concept

Soil Monitoring • Automated Irrigation • Cloud Integration

The project addresses inefficiencies in manual irrigation by introducing an automated solution. The system continuously measures soil moisture, temperature, and humidity, uploads the data to the cloud, and controls a water pump accordingly. This ensures optimized water usage, preventing both under- and over-irrigation.

Hardware Components

Sensors • Microcontroller • Actuators

DHT11 Sensor: Measures air temperature (0–50°C) and humidity (20–90%) with ±1% accuracy.

Resistive Soil Moisture Sensor: Detects soil water content by measuring resistance variation.

ESP32 Microcontroller: A dual-core MCU with integrated Wi-Fi and Bluetooth, programmed via Arduino IDE with FreeRTOS for multitasking.

12V Water Pump + Relay: The relay switches the pump ON/OFF based on soil moisture thresholds received from cloud data.

Cloud Integration

Sensor readings are uploaded to ThingSpeak for visualization. Cloud data allows remote monitoring of temperature, humidity, and soil moisture. Automated logic ensures the pump activates when soil dryness crosses a threshold.

Programming & FreeRTOS

Implemented task-based programming using FreeRTOS:

  • Task 1: Upload temperature and humidity to ThingSpeak
  • Task 2: Upload soil moisture readings
  • Task 3: Control water pump using relay, based on real-time soil conditions
This approach ensures reliable multitasking and smooth real-time operation of all modules.

Final Outcome

The system automatically irrigates crops, displays real-time values on an LCD, and provides live monitoring via cloud dashboard. It is low-cost, power-efficient, and scalable, making it suitable for smart agriculture applications.
